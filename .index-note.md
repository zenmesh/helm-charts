# Index.yaml Note

⚠️ **Important**: The `index.yaml` file must be regenerated from packaged charts - **DO NOT hand-edit it**.

## Current Issue

The `index.yaml` file currently contains stale metadata placeholders (e.g., `github.com/your-org/...`, `<your-gpg-fingerprint>`) from version 1.0.0. These placeholders must be eliminated.

## How to Regenerate

The `index.yaml` file should be regenerated from packaged charts as part of the publishing workflow:

```bash
# Package charts first (if not already packaged)
helm package charts/zen-watcher
# ... package other charts ...

# Regenerate index.yaml from all .tgz files
helm repo index . --url https://kube-zen.github.io/helm-charts
```

This command:
1. Reads all `.tgz` files in the current directory
2. Extracts metadata from each chart's `Chart.yaml`
3. Generates a clean `index.yaml` without stale placeholders
4. Uses metadata directly from the source `Chart.yaml` files (not templates)

## Publishing Workflow (Future)

When automated publishing workflow is implemented, `index.yaml` should be regenerated automatically:
1. Package charts
2. Run `helm repo index . --url https://kube-zen.github.io/helm-charts`
3. Commit the regenerated `index.yaml`
4. Publish to GitHub Pages

**Note**: Chart packages (.tgz files) are excluded from git via `.gitignore`. The `index.yaml` file references charts published to GitHub Pages.
